Data Storage

Overview

The main purpose of the data storage library is to allow you to quickly format, store, and transfer your data to a server. There are two decisions you need to make: (a) what kind of storage you would like to use and (b) what kind of transfer policy you would like to use.

Transfer

The fastest way to implement a storage/transfer handler for your app is to extend one of the abstract classes provided in the com.ubhave.datahandler.loggertypes package.

No Transfer: Extend the com.ubhave.datahandler.loggertypes.AbstractStoreOnlyLogger.java class.

POST Immediately: Extend the com.ubhave.datahandler.loggertypes.AbstractImmediateTransferLogger.java class.

Asynchronous Transfer: Extend the com.ubhave.datahandler.loggertypes.AbstractAsyncTransferLogger.java class.

Secure HTTPS Transfer

Storage

The library provides two kinds of storage, file system, database, and an additional 'no storage' option.

File system: DataStorageConfig.STORAGE_TYPE_FILES. This stores your data in the phone's file system, and requires the android.permission.WRITE_EXTERNAL_STORAGE permission. The data will be stored in directories matching each sensor's name.

SQLite Storage: DataStorageConfig.STORAGE_TYPE_DB. This stores your data in an SQLiteDatabase table that has been named according to the sensor type.

Encrypted SQLite Storage

Combining Storage and Transfer

Note: if you opt to transfer data immediately, you need to set your storage to the 'no storage' option. Similarly, if you only want to store your data (i.e., no transfer) or you want to transfer your data asynchronously, you cannot use the 'no storage' option.
